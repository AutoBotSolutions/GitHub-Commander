## Getting Help

### Before Asking for Help

1. **Read the Documentation**
   - Check the [User Guide](README.md)
   - Review specific guides for your issue
   - Search existing documentation

2. **Search Existing Issues**
   - Check GitHub Issues for similar problems
   - Review closed issues for solutions
   - Search discussions

3. **Check Your Setup**
   - Verify you're using the latest version
   - Ensure dependencies are installed correctly
   - Check system requirements

4. **Gather Information**
   - Note the exact error message
   - Record steps to reproduce
   - Include your environment details
   - Take screenshots if applicable

### How to Ask

When asking for help, provide:

**Environment Information:**
- Operating system and version
- Python version
- GitHub Commander version
- Relevant configuration

**Problem Description:**
- What you were trying to do
- What happened instead
- Steps to reproduce
- Expected vs actual behavior

**What You've Tried:**
- Steps you've already taken
- Solutions you've attempted
- Research you've done

## Reporting Issues

### Bug Reports

Report bugs using GitHub Issues with the following template:

```markdown
**Description**
[Brief description of the bug]

**Steps to Reproduce**
1. 
2. 
3. 

**Expected Behavior**
[What you expected to happen]

**Actual Behavior**
[What actually happened]

**Screenshots**
[If applicable]

**Environment**
- OS: [e.g., Ubuntu 22.04]
- Python Version: [e.g., 3.11]
- GitHub Commander Version: [e.g., 1.0.0]

**Additional Context**
[Any other relevant information]
```

### Priority Labels

- **critical**: Security issues, data loss, crashes
- **high**: Major functionality broken
- **medium**: Minor functionality broken
- **low**: Cosmetic issues, typos

## Security Issues

### Reporting Security Vulnerabilities

**Do NOT** report security vulnerabilities publicly.

For security issues:
1. Send an email to: security@example.com
2. Include "Security Vulnerability" in the subject
3. Provide details about the vulnerability
4. Allow time for response before disclosing

### Security Policy

See [Security Guide](SECURITY.md) for detailed security information.
